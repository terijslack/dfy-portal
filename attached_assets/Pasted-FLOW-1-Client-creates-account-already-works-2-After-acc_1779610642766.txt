NEW ROUTE — POST /api/create-checkout-session:
- Receives: { userId, plan } where plan is 'online-presence', 'growth-engine', or 'done-for-you-partner'
- Creates a Stripe Customer for the user if one doesn't exist, saves stripe_customer_id to DB
- Creates a Stripe Checkout Session in subscription mode using the matching STRIPE_PRICE_* env var
- success_url should be BASE_URL + /payment-success?session_id={CHECKOUT_SESSION_ID}
- cancel_url should be BASE_URL + /signup
- Pass userId and plan in subscription_data.metadata
- Returns { url: session.url } and the frontend redirects to it

MODIFY the account creation route:
- After successfully creating the user account, instead of redirecting to the portal, 
  return the new userId and selected plan to the frontend
- Frontend then calls POST /api/create-checkout-session with userId and plan
- Frontend redirects to the Stripe URL returned

NEW ROUTE — POST /webhook/stripe:
- This route must use express.raw({type: 'application/json'}) middleware, NOT express.json()
- Verify the webhook signature using STRIPE_WEBHOOK_SECRET
- Handle these events:
  * checkout.session.completed → update user: subscription_status='active', 
    stripe_subscription_id=session.subscription, subscription_start_date=NOW()
    Get userId from event.data.object.metadata or subscription_data.metadata
  * customer.subscription.deleted → update user: subscription_status='cancelled'
  * invoice.payment_failed → update user: subscription_status='past_due'
- Return { received: true }
